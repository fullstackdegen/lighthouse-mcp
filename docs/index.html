How it works

From score to fix pack

A Lighthouse score of 67 doesn't tell your agent which component to edit. Agent Audit converts each failing audit into a fix pack with the evidence, the search hints, and the acceptance criteria built in.

fix pack · accessibility · priority 2
{
  "id": "fix-link-name",
  "severity": "critical",
  "goal": "Fix links that do not have a discernible name",
  "repoSearchHints": [
    "div.border-t-2 > div.flex > a.text-gray-600",
    "https://www.linkedin.com/company/commalabs"
  ],
  "implementationSteps": [
    "Search repo using repoSearchHints before editing",
    "Add aria-label or visible text to each bare link"
  ],
  "acceptanceCriteria": [
    "All links pass the Lighthouse link-name audit",
    "Median accessibility score ≥ 90"
  ]
}

The CSS selectors in repoSearchHints come directly from Lighthouse's failing audit evidence — your agent uses them to locate the component without guessing.

Coverage

What gets audited

Each run covers mobile and desktop profiles. Reliable mode runs three times per profile and returns median scores with variability ranges.

  • FCP, LCP, CLS, TBT, Speed Index
  • Canonical tags and robots signals
  • Accessibility — WCAG issues with selectors
  • JSON-LD structured data
  • Broken links and missing link names
  • Open Graph tags
  • Image alt text and sizing
  • LLM crawler visibility
  • Indexability signals
  • llms.txt readiness
